## Prompt Tuning Recommendations

### 1. inbox-triage.md — Add Batch Size Guardrail
**Problem:** Graph API timeouts when processing >50 emails (session `8bf92077`). Turn 2: user noted "this seems to take a long time."
**Suggested change:** Ensure Rule #4 in inbox-triage.md specifies batch sizes of 5 for delete/archive operations, and add a rule to paginate email fetches to max 25 per page.
**Expected impact:** Eliminates API timeout failures during triage.

### 2. copilot-instructions.md — Add poc-scoper Discovery Hint
**Problem:** In session `28043a2f`, the user had to explicitly ask "i think we have a poc scoping agent to do this for us right?" — the copilot didn't suggest it.
**Suggested change:** Add to `copilot-instructions.md`: "When the user asks to create a brief, scope a POC, or compile requirements for a customer engagement, suggest using the `poc-scoper` agent."
**Expected impact:** Faster routing to the right agent without user prompting.

### 3. email-voice skill — Solidify the "No Dashes" Rule
**Problem:** In session `9bd24935`, multiple iterations were needed to get email formatting right. The user explicitly said "NO - DASHES" at turn 10.
**Suggested change:** The `email-voice` skill was created during this session. Verify it includes an explicit rule: "Never use em dashes (—), en dashes (–), or hyphens (-) as separators or stylistic elements in email prose."
**Expected impact:** First-pass email drafts match Harry's writing style.

### 4. All Agent Definitions — Add YAML Frontmatter Validation Note
**Problem:** Session `802f8f07` — agent broke due to embedded quotes in YAML.
**Suggested change:** Add a comment at the top of each agent .md file: `<!-- YAML frontmatter: avoid embedded quotes in string values. Use single quotes or escape with backslash. -->`
**Expected impact:** Prevents future YAML parse breakages when editing agent definitions.
