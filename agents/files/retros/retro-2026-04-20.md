## Friction Points

### 1. Agent Definition Fragility (Sessions 5, 6)
**Impact: HIGH** — 2 of 10 sessions spent debugging agent loading issues.
- Session 6: an agent file had embedded double quotes in YAML frontmatter that broke parsing. The first fix attempt was wrong; the second found the real bug.
- Session 5: an MCP server referenced in frontmatter wasn't configured, causing confusion before the missing dependency was identified.

**Root cause:** No validation step when editing agent definitions. YAML frontmatter is brittle.

### 2. Self-Tooling Overhead (Sessions 5, 8)
**Impact: MEDIUM** — ~20% of sessions went to environment/tooling rather than customer work (MCP config, hotkey shortcut). Worth a one-time cleanup pass.

### 3. QB Pipeline Under-Exercised Historically
**Impact: STRATEGIC** — Session 7 was the first real end-to-end QB run in this window. It went well, but a single data point isn't enough. Run more low-stakes tasks through QB to build confidence in the gates and iteration protocol.

## Prompt Tuning Recommendations

### 1. scoper — Add Discovery Hint
**Problem:** In an earlier session the user had to explicitly ask whether a scoping agent existed.
**Suggested change:** Add guidance so that requests to "create a brief / scope a POC / compile requirements" route to `scoper` automatically.
**Expected impact:** Faster routing to the right agent without user prompting.

### 2. All Agent Definitions — Add YAML Frontmatter Validation Note
**Problem:** Agents broke due to embedded quotes in YAML (session 6).
**Suggested change:** Add a comment to each agent `.md`: avoid embedded quotes in string values; use single quotes or escape with a backslash.
**Expected impact:** Prevents future YAML parse breakages when editing definitions.

### 3. QB — Surface Quality-Gate Evidence
**Problem:** Session 7's gates caught regressions, but the "why" wasn't summarized for the user.
**Suggested change:** Have QB report a short evidence line per bounce (what failed, what fixed it).
**Expected impact:** Clearer audit trail and faster human approval at checkpoints.

## Workflow Recommendations

### 1. Force-Test the QB Pipeline Regularly
Pick a low-stakes task each week and run it through QB end-to-end so the quality gates, sub-agent routing, and iteration protocol stay exercised and trustworthy.

### 2. Add a "Validate Agent YAML" Utility
A simple script that validates all agent `.md` frontmatter before load would have caught both issues in sessions 5 and 6 instantly. Run it after any agent edit.

### 3. Batch Self-Tooling Work
Group environment/MCP/hotkey fixes into a single maintenance session rather than interrupting customer work.

## New Skill / Agent Suggestions

### 1. `yaml-validate` Skill
Validates YAML frontmatter in all agent/skill definition files. Catches embedded quotes, missing required fields, and invalid tool references. Run as a pre-flight check after any agent edit session.

### 2. `workload-report` Skill
Session 3 showed demand for "compile all my POC asks into a visual." A repeatable skill could aggregate by customer/status/priority and output a formatted report.
